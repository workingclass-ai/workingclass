# 红旗扫描器 / Red-Flag Scanner

> **怎么用**：60秒过一遍。每一条都用"过去30天"作为窗口。√打勾，× 不适用。
> 总分>=8 → 你的处境需要严肃应对。建议立刻进入 `/pip` 模块第一阶段 + 启动秘密求职（`/jobsearch`）。

---

## A. 公司层面信号（每条 1 分）

- [ ] 公司刚宣布裁员、招聘冻结、或 reorg
- [ ] 公司刚发了不好的财报 / 融资失败 / 估值下调
- [ ] 你的 BU / 团队 / 产品线被高层公开质疑或"under review"
- [ ] 高管最近频繁离职（CEO/CFO/VP 级别）
- [ ] 公司开始大量发"我们要更 lean / efficient / focused"的内部信
- [ ] 你的公司刚被收购，或宣布要被收购
- [ ] 同部门有人最近被裁，且没补人

## B. 你和经理的关系（每条 1 分）

- [ ] 1:1 频率突然变了（变密 = 在记录证据；变稀 = 在脱钩）
- [ ] 经理开始用邮件/书面方式给你"建设性反馈"，而以前是口头
- [ ] 经理在邮件里抄送 HR、skip-level、其他经理（建立 paper trail）
- [ ] 经理问你"职业规划"、"你 long-term 怎么想"——但不是 promo 语境
- [ ] 经理对你以前赞过的工作开始挑刺
- [ ] 你和经理之间出现"误会"，但你解释完他没真的接受
- [ ] 经理推迟你已经口头确认过的升职/加薪/股票
- [ ] 经理开始用模糊词："ownership"、"leadership"、"impact"——但拒绝具体定义

## C. 工作内容信号（每条 1 分）

- [ ] 你被从核心项目移到边缘项目
- [ ] 新启动的项目开始绕过你
- [ ] 你曾经的 stakeholder 被重新分配给别人
- [ ] 你的代码 review / 设计文档收到的反馈突然变挑剔
- [ ] 你被要求做大量"知识转移 / 文档化"工作（特别是 AI 时代——你在训练替代你的东西）
- [ ] 你被分配的工作量明显减少（在为不需要你做铺垫）
- [ ] 你被分配的工作量明显增加 + 不可能完成的 deadline（在为"绩效不足"做铺垫）
- [ ] 同事/下属对你的态度有微妙变化——疏远、不再 cc 你

## D. HR / 流程信号（每条 1 分，权重高）

- [ ] HR 突然加入了你的 1:1（**这条 +2 分**）
- [ ] 你被要求做 360 review 或 calibration——但不是常规节奏
- [ ] 你的权限被静默回收：系统访问、文档权限、会议邀请、Slack channel
- [ ] 你被告知"我们要做一次正式的 performance evaluation"
- [ ] 你刚拿到一份比你预期低的绩效review，但没有"明确的下一步"
- [ ] 你被叫进 HR 主持的会议，议题模糊
- [ ] 公司启动了大规模的 calibration / stack ranking 流程

## E. 法律保护类信号（**任何一条出现都立即找律师**，不计分）

- [ ] 你最近请了病假、医疗假、产假、parental leave
- [ ] 你最近举报过 harassment、安全问题、合规问题、财务问题
- [ ] 你最近申请了合理便利（reasonable accommodation）或工伤
- [ ] 你属于年龄/性别/种族/性取向/残疾保护类别 + 上面信号叠加
- [ ] 你刚拒绝了上司的不合理要求（包括但不限于不当性接触、违法操作）
- [ ] 你最近投诉过工资/加班费问题

## 评分

| 总分 | 状态 | 建议 |
|------|------|------|
| 0-2 | 绿色 | 你大概率在多想。继续观察，正常工作。|
| 3-5 | 黄色 | 有信号但不确定。建议：把过去30天的事件用**时间线方式**记下来，私人备份。继续观察。开始非紧急地更新简历。|
| 6-8 | 橙色 | 多个信号叠加。建议：进入 `/pip` 模块第一阶段。开始**主动**而不是被动地求职——告诉信任的人你"在看机会"。开始记录所有书面反馈。|
| 9+ | 红色 | 高概率你处在 PIP 前夜或裁员前夜。立刻：(1)把所有正面证据备份到个人邮箱（见`references/layoff-first-72-hours.md` 第1小时清单），(2)启动秘密求职（`/jobsearch` 9A），(3)了解你的 severance 杠杆（`/pip` 第三阶段C），(4)如果有 E 区任何一条信号，找雇佣律师。|

## 重要原则

**红旗不是判决书**。出现 6 个信号不意味着你 100% 要被裁——意味着你处在**应该有备份计划**的处境。备份计划不需要触发——但如果不备份，等触发时已经晚了。

**不要把扫描结果告诉经理或 HR**。他们不会因为你"发现了"就停止流程；只会因为你太快暴露准备而失去你的杠杆。

**不要在公司设备上做这个扫描**，也不要保存在公司 Google Drive / OneDrive / Slack 里。

---

## 反向扫描：你"以为有信号但其实没有"的情况

为了避免过度警觉，下面这些**不是**red flag：

- 经理某次1:1表达了一些建设性反馈 —— 这是经理的工作
- 你某个项目失败了 —— 单一事件不是模式
- 你没拿到升职 —— 升职数量受限不等于你被针对
- 公司宣布裁员但你的 BU 没被点名 —— 关注但不恐慌
- 你和某个同事关系变差 —— 单点摩擦不是系统信号

红旗的关键是**多点同时出现** + **方向一致**（都指向"减少对你的投入 / 建立可解雇你的证据")。单点信号几乎没有意义。
