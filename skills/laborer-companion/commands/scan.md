# 红旗扫描 / Red-Flag Scan

## 你的任务

读取 `references/red-flag-scanner.md`，把信号清单**逐项**问用户（按 A→B→C→D→E 的顺序），最后给出评分和对应建议。

## 步骤

### Step 1：读完整清单

```
references/red-flag-scanner.md
```

### Step 2：开场提醒

先告诉用户：

> 我们做一个 60 秒的扫描。我会按 5 类问 30 多条信号，你只需回答**有 / 没有 / 不确定**。
>
> **过去30天作为窗口**——只看最近一个月发生的事。
>
> 这个扫描的目的不是给你判决书，是帮你识别需不需要做备份计划。准备好了吗？

等用户确认后开始。

### Step 3：分类提问

按 A（公司）→ B（你和经理）→ C（工作内容）→ D（HR/流程）→ E（法律保护）的顺序。

每一类一次问完——不要一条条问，那太累。例如：

> **A. 公司层面 - 过去30天里，下面任何一条出现过吗？（多选，简单回是哪几条就行）**
>
> 1. 公司刚宣布裁员、招聘冻结、reorg
> 2. 公司刚发了不好的财报 / 融资失败
> 3. 你的 BU/团队/产品线被高层公开质疑
> 4. 高管最近频繁离职
> 5. ...

### Step 4：计分

按 references/red-flag-scanner.md 的规则：
- A、B、C、D 区每条 1 分
- D 区"HR突然加入1:1"那条 +2 分
- E 区不计分但任意一条出现都要单独提示

### Step 5：输出

```markdown
## 你的扫描结果

**总分**：X / Y
**状态**：🟢绿 / 🟡黄 / 🟠橙 / 🔴红

### 关键信号（你勾选的）
- [信号1]
- [信号2]
- ...

### E 区警告（如果有）
[出现的法律保护类信号 → 强烈建议立刻找雇佣律师]

### 我的判断
[2-3 句话，直接说现在的处境是什么]

### 建议（按状态分）

🟢 → ...
🟡 → ...
🟠 → ...
🔴 → 立刻做这4件事：(1)备份证据 (2)启动秘密求职 (3)了解 severance 杠杆 (4)看下面下一步

### 下一步
- [具体的命令调用建议，比如 /pip 或 /jobsearch]
- [是否需要看 references/layoff-first-72-hours.md]
- [是否需要看 references/privacy-and-opsec.md（如果用户即将开始备份证据）]
```

## 重要原则

1. **不要让评分听起来像判决**——红色不等于100%要被裁，是"应该有备份计划"的状态
2. **遇到 E 区任何信号立刻强调找律师**——不要等扫描结束
3. **不要让扫描超过 5 分钟**——这是入口工具，不是治疗
4. **扫完之后路由到具体模块**——不要在这里展开分析
