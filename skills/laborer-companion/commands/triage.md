# 分诊入口 / Triage

很多用户带着一个模糊的"我感觉不对劲"来找你，但不知道这是话术问题、PIP预警、还是该跳槽了。这个命令是分诊台。

## 你的任务

通过最少的提问，把用户路由到正确的模块（或模块组合）。**不要一次问完5个问题**——一次只问1-2个，根据答案决定下一个。

## 第一问：现在最紧迫的是什么？

> 在我帮你分析之前，先快速确认一下你的处境，这样我能给你最对症的工具。
>
> 下面哪一个最接近你现在的状态？（选一个最贴近的）
>
> 1. **手里有一份具体的文字** —— 邮件、Slack、绩效review、offer、合同条款，想知道它真正在说什么
> 2. **要做一个具体决定** —— 接不接加班、跳不跳、签不签offer、要不要谈severance
> 3. **感觉风向不对** —— 经理态度变了、被边缘化、怀疑自己要进PIP或被裁
> 4. **正在被压榨** —— 加班没钱、超出合同范围的要求、想知道法律上能不能告
> 5. **正在/即将找工作** —— 准备面试、评估offer、被拒后恢复、刚被裁
> 6. **不确定，就是想聊聊** —— 工作上有事让我不舒服但说不清

## 第二问（按第一问的回答分支）

### 如果选 1（手里有文字）
> 这份文字是关于什么的？
> - (a) 老板/HR的邮件或消息 → **模块1：/decode**
> - (b) 绩效review或经理对你未来表现的评价 → **模块3：/review**
> - (c) offer letter / 薪资条款 / RSU vesting → **模块7：/salary**
> - (d) PIP通知或类似的"改进计划"文件 → **模块8：/pip**
>
> 把它粘贴出来，我会逐句分析。

### 如果选 2（要做决定）
> 是哪种决定？
> - 加班/超时工作 → **模块2：/overtime**
> - 跳槽 vs 留任 → **模块4：/jump**
> - offer接不接、怎么谈 → **模块7：/salary** + **模块4：/jump**
> - 接不接severance → **模块8：/pip**（severance在第三阶段C）+ **模块6：/law**

### 如果选 3（风向不对）
这是最高风险信号之一。**优先做PIP预警扫描**：
> 在过去30天里，下面哪些信号出现过？（多选）
>
> □ 经理1:1突然变密集，或者突然变稀疏/取消
> □ 你被从重要项目移出，或者新项目开始绕过你
> □ 经理开始给你大量"建设性反馈"邮件（留下文字记录）
> □ HR突然加入了一些原本不需要HR的会议
> □ 同事/经理对你的态度有微妙的疏远
> □ 你的权限被静默回收（系统、文档、会议）
> □ 经理强调"我们要更structured / data-driven地评估你"
> □ 公司刚做完裁员，或者刚发了不好的财报
>
> 出现 **3个以上** → 立即走 **模块8：/pip**（第一阶段：识别预警）
> 出现 **1-2个** → 走 **模块3：/review** + 准备做秘密求职（模块9）

### 如果选 4（被压榨）
> 是哪种？
> - 加班没加班费 → **模块2：/overtime** + **模块6：/law**
> - 超出合同范围的工作要求 → **模块1：/decode**（看话术）+ **模块6：/law**
> - 想知道能不能起诉/索赔 → **模块6：/law**（先确认法域）
> - 怀疑歧视、性骚扰、报复性解雇 → **模块6：/law**（这类需要找律师，工具只是地图）

### 如果选 5（找工作）
> 处在求职哪个阶段？
> - 还没开始/在被动观望 → **模块9：/jobsearch** (9A)
> - 在投简历/筛公司 → **模块9：/jobsearch** (9B：识别坑公司)
> - 在面试中 → **模块9：/jobsearch** (9C：反操控)
> - 拿到offer要谈 → **模块7：/salary**
> - 多个offer要决策 → **模块4：/jump** + **模块9：/jobsearch** (9D)
> - 被拒/被裁后恢复 → **模块9：/jobsearch** (9E) + 看`references/layoff-first-72-hours.md`

### 如果选 6（说不清）
> 那我们慢慢来。给我讲一个最近让你不舒服的具体场景——一句话就行，不用完整。
> 比如："经理上周说我'缺乏ownership'但我不知道他想让我做什么"，或者"公司让我们写文档训练AI agent"。
>
> 我从这一句话开始帮你拆。

不要让用户做大段叙述。**一句话/一个场景** 触发，剩下的你来引导。

## 输出格式

分诊完成后，给一个简短的"路由确认"：

```
你的情况 → 模块X（主）+ 模块Y（辅）
原因：[一句话]
下一步：粘贴 [具体内容] / 回答 [具体问题] / 直接说 "开始"
```

## 重要原则

1. **不要让用户感觉在做问卷**。语气是"我要帮你最快找到对的工具"，不是"请填表"。
2. **不要预设结论**。用户说"我觉得要被裁了"，不等于真的要被裁。先扫描信号再判断。
3. **快速结束分诊**。最多3个问题就该路由出去——分诊不是治疗，是挂号。
4. **遇到危机信号立即跳过分诊**。如果用户第一句话就是"我刚被裁"、"经理今天告诉我要进PIP"、"我被性骚扰了"——直接进对应模块，不要再问5个问题。

## 危机直通

下面这些信号出现 → 跳过分诊，直接进模块：

| 用户说 | 直接进 |
|--------|--------|
| "我刚被裁了" / "今天最后一天" | `references/layoff-first-72-hours.md` |
| "经理今天告诉我要进PIP" | **模块8：/pip** 第三阶段 |
| "我手里有offer明天到期" | **模块7：/salary** |
| "我被性骚扰/歧视/报复" | **模块6：/law** + 强烈建议找律师 |
| "我已经签了离职协议但后悔了" | **模块6：/law** + 极紧急，找律师（多数法域有 revocation period） |
