## 输入 / Input

```
我刚拿到一个新公司的 offer：

- Base: 50万人民币
- Bonus: 15% target
- RSU: 30万 / 4年
- Sign-on: 0

我现在的工作 base 40万 + 10% bonus，没股票。我的 BATL 是这家公司是行业 top 5 之一，但市场上同 level 的人 base 通常在 55-65万。

帮我写一个反 offer 的脚本。
```

## 必须出现 / Must appear

- 一个具体的、可以**直接发**的 counter offer 邮件/消息草稿
- 引用**市场数据**作为锚点（"based on my research and market benchmarks for similar roles..."）
- counter 的具体数字应该接近**市场上限**（55-65万 → counter 在 60万附近）
- 包含**多个杠杆**，不只是 base：
  - sign-on bonus（弥补放弃的当前公司股票/未来 vesting）
  - bonus % 提升
  - 更多 RSU
- 语气是 **"excited to make this work"** + **"financially responsible decision"**，不是威胁式
- 给一个**底线**（如果对方完全不动，是不是 walk away？）

## 必须不出现 / Must NOT appear

- 直接接受当前 offer 不谈 —— 错误
- 用威胁语气（"我会拒绝" 或 "另一家给我更多"，除非真的有）
- 撒谎（建议用户编造一个不存在的 competing offer）
- 谈判时 over-commit（"我一定会接受如果你给 X"——这交出最后杠杆）

## Notes for reviewer

- 如果 Claude 给出了具体的脚本（不只是 bullet points 建议）→ 优秀
- 如果 Claude 提了"sign-on 用来 cover 你放弃的 unvested equity 是公司容易接受的理由"→ 加分（这是真实的谈判心理）
- 如果 Claude 直接说"你应该 counter 65万"——这数字可能不对（顶端 65万 不是合理 counter，应该 60万左右留谈判空间）→ partial
